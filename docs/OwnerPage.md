# 主催者用募集情報について

## 募集情報

主催者相当の権限者が確認できる募集情報は以下の情報を持つ

### 募集のID

各募集に割り当てられるULID、自動で割り当てられるので主催者が編集することはできない

### 募集編集に利用するパスワード

主催者は任意のパスワードを募集作成時に設定でき、募集編集者を制限できる

パスワードを設定しない場合、募集ページのURLを知っている人間すべてを主催者と同じ扱いにする

利用前提は身内限定とするので不正な編集は考慮しない

### 主催者名

主催者がテキストボックスに入力する名前

ユーザー情報などと紐づけて管理することはない

### 募集概要

募集についての概要

会で何を行うかなどを記述する

日時などの情報は書かない事を推奨する

### 募集場所

会場名をテキストで入力

オンライン開催の場合のURLは事前に作成できる場合はここに記述、事前作成が難しい場合は主催者が別途Discordで送信する

### 募集期日

主催者は募集の期間を設定することができ、カレンダービューで募集終了日時を設定する

募集開始日に関しては、作成日を適用する

### 募集状況

下記4つの状態があり、`OPEN -> EXPIRED`は自動で遷移し、`CLOSED` `DELETED`は主催者が手動で遷移させる

- `OPEN`: 募集中
- `CLOSED`: 募集確定
- `EXPIRED`: 募集期限終了
- `DELETED`: 募集削除済み

#### 状況遷移について

- 状況は上記募集期日内だと`OPEN`

- 募集終了日時になると`EXPIRED`へ遷移し、主催者が日時を確定するまで継続する

- 主催者が日時を確定すると`CLOSED`へ遷移する (募集が`EXPIRED`状態でなくとも、主催者の操作があれば`CLOSED`にすることは可能である)

- 主催者が削除操作を行った場合`DELETED`へ遷移し、遷移後1週間で削除される

### 募集日時候補一覧

MM-dd(dayofweek)HH:mm (05-19(火)19:00) 形式の日時データの一覧を表で表示する

#### 募集日時の登録について

内部ではdatetimeで扱うので、登録時はカレンダービューやテキストでMM-dd HH:mm形式で登録し、曜日は表示時に自動計算、

年は募集作成年、タイムゾーンはAsia/Tokyoでdatetime型へ変換する

原則年は募集作成年だが、変換後の候補日時の月日が募集作成日時の月日より前になる場合は、作成年+1年として扱う

2年先などは考慮しない

### 参加者表

Rowを参加者名、Columnに日時候補を設定し、末行に日時候補ごとの参加可能人数の集計を表示する

### Discordリマインドボタン

`OPEN`時のみ主催者の任意のタイミングでDiscordへリマインドメッセージを送信することができる
