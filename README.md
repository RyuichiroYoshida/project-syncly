# project-Syncly

## 概要

コミュニティ内予定調整サービス

## 募集の流れ

1. ブラウザ上で主催者が日時候補と会の概要を記載した募集を作成する
2. 作成後、上記内容がDiscordへ送信される
3. 参加者は主催者が提示した候補内で自身の希望日とリモート参加を行うかの意思表示、コメントを名前付きでブラウザ上で入力する
4. 日時確定後、日時確定の旨と募集内容のリマインドがDiscordへ送信される
5. 参加者は確定された日時、確定された参加者を確認できる

## MVP

- 参加者は主催者が募集作成時に提供する日時の候補、リモート参加の可否、コメントに回答できる
- 主催者は日時募集終了後、又は任意のタイミングで日時を確定する
- 募集への回答は参加者表として募集ページに記載され、主催者及び参加者が確認することができる
- 参加者はDiscordで確定された募集の日時、場所を知ることができるが、詳細や他の参加者などはWeb上の募集ページを確認する必要がある

## ドメイン

- 参加者一覧(人数)が必要
- 主催者が日時を登録する
- 募集をDiscordへ送信する
- ブラウザ上で募集は削除でき、募集を削除した際にはDiscordへ募集削除の通知を送信する
- 募集中かの状態を確認できる

## 各情報

### 募集情報

[docs/OwnerPage.md](./docs/OwnerPage.md) へ記載

### 参加者情報

厳密にユーザー管理は行わない

- 名前 (同一募集内で重複不可)
- 募集日時内での参加候補日時
- リモートの可否
- コメント

#### 参加者情報の編集

各参加者は募集が`OPEN`の状態の場合のみ、各々が設定した名前をクリックすると回答を編集できる

他人の回答も編集することはできるが、利用前提は身内限定とするので考慮しない

### 募集ページ

募集IDごとにURLを発行し、募集一覧表示などは存在しない

## 技術選定について

### Discord

- Webhookを利用し、募集作成時、募集確定時、募集削除時に通知を送信する
- 募集終了24時間前リマインドは募集が`OPEN`時のみ送信する
- 募集`OPEN`時のみ主催者の任意のタイミングで通知を送信できる

### フロントエンド

- Next.js / TypeScript

### バックエンド

API本体、ドキュメント
